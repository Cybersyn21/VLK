# Анализ папки TEXT - План обработки

## Что находится в папке TEXT

Согласно анализу GitHub репозитория, папка `TEXT` содержит **67 файлов**:

- **Основной контент**: `.doc` и `.docx` файлы со стихотворениями Владимира Волкова
- **Аудиофайлы**: `.mp3` файлы с записями некоторых стихотворений
- **Метаданные**: `.json` файлы с информацией об аудио
- **Разметка**: `.html` и `.txt` файлы

### Примеры файлов:
- "Каждой ночью на болоте" (.doc, .mp3, .mp3.json)
- "Зазвенят две натянутых рядом струны" (.docx, .mp3, .mp3.json, .txt)
- "Кругом учёные мужи" (различные форматы)

## Что нужно сделать (мои догадки)

Исходя из предыдущей работы над проектом **VOLKOV2.0**, где мы создали структурированные JSON и TXT файлы для 96 стихотворений из 5 альбомов, логично предположить следующие задачи для папки TEXT:

### 1. Извлечение и структурирование текстов
- Извлечь тексты из всех `.doc` и `.docx` файлов
- Очистить от форматирования и HTML-тегов
- Разделить на строфы (пустые строки между ними)

### 2. Создание JSON структуры
Для каждого стихотворения создать JSON файл по аналогии с VOLKOV2.0:
```json
{
  "title": "Название стихотворения",
  "link": "ссылка на аудио (если есть)",
  "text": [
    "строка 1",
    "строка 2",
    "",
    "строка следующей строфы"
  ]
}
```

### 3. Связь с аудиофайлами
- Для стихотворений с `.mp3` файлами - добавить ссылки на аудио
- Проверить метаданные из `.json` файлов
- Возможно, залить аудио на сервер (аналогично v-volkov.ru/audio/)

### 4. Проверка орфографии и пунктуации
Как мы делали с другими стихотворениями:
- Проверить "ё" вместо "е" (придёт, берёг, и т.д.)
- Проверить слитное/раздельное написание (кто-то, что-то, навряд ли)
- Проверить пунктуацию (запятые при обращениях, вводных словах)
- Проверить "по среди" → "посреди" и подобные ошибки

### 5. Создание единой структуры
```
TEXT_PROCESSED/
├── 01_название_стихотворения.json
├── 01_название_стихотворения.txt
├── 02_название_стихотворения.json
├── 02_название_стихотворения.txt
├── ...
├── audio/ (если есть локальные mp3)
│   ├── 01_название.mp3
│   └── ...
├── ssilki.txt (список всех аудиоссылок)
└── README.md
```

### 6. Создание скриптов обработки
Аналогично VOLKOV2.0:
- `extract_from_doc.py` - извлечение текстов из .doc файлов
- `create_json_files.py` - создание JSON файлов
- `verify_spelling.py` - проверка орфографии
- `link_audio.py` - связывание с аудиофайлами

## Возможные проблемы

1. **Формат .doc файлов** - могут содержать сложное форматирование
2. **Дубликаты** - некоторые стихотворения могут пересекаться с VOLKOV2.0
3. **Названия файлов** - длинные, с нестандартными символами
4. **Аудио** - не для всех стихотворений есть записи
5. **Метаданные** - нужно понять структуру .json файлов

## Транскрибация аудио

**К сожалению, я не могу транскрибировать аудиофайлы**, так как не имею возможности обрабатывать аудио. Для транскрибации аудио в текст потребуются специальные инструменты:

- **Whisper** от OpenAI (open-source)
- **Google Speech-to-Text API**
- **Yandex SpeechKit**
- Или другие сервисы распознавания речи

Если в файлах `.mp3.json` уже есть транскрипты, их можно использовать.
